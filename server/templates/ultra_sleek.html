🛰️ CO-BRAIN OPERATOR MANUAL v1.1

Purpose: This browser simulation is an interactive NISQ operations cockpit. You tune error sources, run a simulated experiment, and observe how mitigation changes leakage and fidelity in real time.


🧭 HOW TO USE THE SIMULATION:
  • Step 1: Set Noise Amp and Gate Drift with either sliders or numeric inputs.
  • Step 2: Choose mitigation mode with Pauli-Twirling (ACTIVE/BYPASS).
  • Step 3: Click ⚡ RUN NISQ EXPERIMENT to compute hardware state, leakage, and fidelity.
  • Step 4: Compare outcomes across settings, then export with 📁 Download Snapshot.
🔘 BUTTON & CONTROL DEFINITIONS:
  • 🌌 Go Sleek / 📟 Retro Mode: Switches between visual themes; simulation logic and backend state remain the same.
  • ❓ Operations Manual: Opens this operator guide popup.
  • [CLOSE MENU]: Closes the manual popup.
  • 📁 Download Snapshot: Exports current state from /api/state as JSON for record-keeping/comparison.
  • Wipe Session: Clears authenticated operator session and signs out.
  • Noise Amp (slider/number): Controls decoherence intensity. Higher values increase stochastic disturbance.
  • Gate Drift (slider/number): Controls gate calibration error drift (%-style value), increasing coherent control error.
  • Pauli-Twirling (ACTIVE/BYPASS): Sends mitigation toggle to /api/tune; ACTIVE randomizes coherent error into more manageable stochastic channels.
  • ⚡ RUN NISQ EXPERIMENT: Calls /api/simulate and updates Hardware State, Matrix Leakage Rate, Fidelity, and Insight.
⚛️ QUANTUM COMPUTING MECHANICS (WHY THIS UI MATTERS):

Classical bits are 0 or 1. Quantum bits (qubits) use superposition, allowing a state to hold weighted probability amplitudes across 0 and 1 simultaneously. Multi-qubit entanglement introduces correlations unavailable to classical systems. Quantum gates rotate these amplitudes, and measurement collapses them to classical outcomes.

Real hardware is noisy: interactions with the environment and imperfect controls introduce decoherence, drift, crosstalk, and readout error. This simulator focuses on that practical layer by mapping your tuning inputs to leakage/fidelity behavior under mitigated vs unmitigated conditions.

🧪 WHY NISQ IS THE MOST RELEVANT TOPIC TODAY:

NISQ (Noisy Intermediate-Scale Quantum) devices are the quantum computers available now: useful for experimentation, benchmarking, hybrid workflows, and error-mitigation research, but not yet fully fault-tolerant. Because large-scale error-corrected machines are still emerging, the most valuable current work is understanding noise behavior, improving calibration, and developing mitigation strategies like Pauli twirling. This console is designed around that exact operational reality.
